The shell tool runs bash and accepts multi-line scripts on stdin; commands
need not be chained on a single line with `&&` or `;`. It trims stdout
and stderr, returning only the first 2kB and last 8kB of each stream.
Multiple calls issued together run in parallel, and exec can be used to
switch interpreters for the remaining input, e.g. `exec python`.

Inspect files before changing them. Use `wc -l FILE...` to check file
lengths, and `cat -n FILE` to view content with line numbers, piping
output into `head`, `tail` or `sed -n START,ENDp` for large files.

`cat >FILE` rewrites a file, and `cat >>FILE` appends to it, but prefer
unified diffs for surgical edits:
```
exec git apply --no-index --recount --unidiff-zero
--- a/FILE
+++ b/FILE
@@ -67,4 +67,4 @@
 context above
 more context
-old line
+new line
 context below
```
Here, `exec` replaces bash with the command, passing remaining stdin
directly. Line numbers in `@@` hunk headers need only be approximate and
counts are corrected by `--recount`. Context lines need a leading space
even if blank. A simple replacement can have just - and + lines, but it
matches exactly once, closest to the specified start line. Changes are
applied atomically; no file is modified if any hunk fails.

`sed -i` and `awk` also work well for mechanical, pattern-based changes.
